# Design Contract — v0.8.0

**Read this before building ANY report. Every rule here is mandatory.**

The skill produces reports with one visual language shared across every report type. Monthly review, weekly summary, hygiene audit, campaign leaderboard, forecast, lead profile, anomaly detector — all 18 reports look like they came from the same designer, in the same week, for the same client. If two reports are opened side-by-side and feel like they were made by different hands, the contract was violated.

## Architecture: one template, many reports

Every report is generated by populating `templates/report-shell.html` with a `sections` array in its embedded JSON. You do not write HTML from scratch. You do not make per-report templates. You do not invent new section types.

The summary JSON schema is:

```json
{
  "meta": { ... },
  "sections": [
    { "type": "title-block", ... },
    { "type": "stat-strip", ... },
    { "type": "chart-insight", ... },
    ...
  ]
}
```

The template iterates `sections` in order and renders each using its `type`. See `scripts/template-schema.md` for the full section reference.

## The locked visual language

### Two-tone cards (the house style)

Every container in the report uses a two-tone split:
- **Top zone**: tinted with the brand surface color. Holds the kicker + the primary content (number, title, action label).
- **Bottom zone**: white. Holds the secondary content (label, body text, delta).
- **Hairline border** (1px rgba(11,27,52,0.10)) around the whole card.
- **4px rounded corners** on the outer container. No soft-heavy radius.
- **No accent bars** (no top, left, right, or bottom color stripes). The brand color appears in text (kickers, italic accents, label text) — not as a decorative stripe.
- **No shadows**. No drop shadow, no elevation, no glow.

Applies to: `stat-strip`, `chart-insight` (the insight card), `insight-card`, `recommendations` (each card), `hero-number`.

### Typography — three fonts, strict usage

- **Instrument Serif** (display): titles, section headers, all numbers, "hero" values
- **Geist** (body): all paragraph copy, chart axis labels
- **Geist Mono** (mono): kicker labels (uppercase, letter-spaced), stat labels, delta values, small annotations

Numbers are always **italic Instrument Serif**. This italic-numeral choice is a core part of the look. Don't use italic for any other purpose (only for the italic-accent word in titles, see below).

### Italic accents in titles

Every slide/section title has ONE word (or short phrase) wrapped in asterisks — the renderer converts it to an italic serif fragment in the brand primary color:

- `"Your *attribution* picture."` → *attribution* is italic, brand-primary
- `"Top UTM *sources*."` → *sources* is italic, brand-primary
- `"Where your *attribution* is leaking."` → *attribution* is italic, brand-primary

Use one italic accent per title. Pick the semantically most important word. This is how the brand color shows up in typography, not as backgrounds.

### Color palette — five tokens

Every color in every report comes from five CSS variables:

```css
--b-primary:  /* brand primary — headlines, kickers, italic accents, chart bar #1 */
--b-ink:      /* near-black for body text and stat numbers */
--b-accent:   /* brand secondary — chart bar #2, rare emphasis */
--b-muted:    /* WCAG-AA grey for secondary text, labels */
--b-surface:  /* very pale tint for the top zone of every two-tone card */
```

These are overridden per-brand at render time (see `references/branding-profile.md`). A fresh install defaults to UTM Grabber's blue palette. `/brand` reconfigures them to the customer's brand.

### Chart palette

```
1. var(--b-primary)
2. var(--b-accent)
3. #FFC857  (warm yellow — brand-agnostic)
4. #30B47A  (green — brand-agnostic)
5. #6FB1E8  (pale blue — brand-agnostic)
6. #F59E0B  (amber — brand-agnostic)
7. var(--b-ink)
```

Used in order. Never introduce additional per-report colors.

### Delta indicators

Unicode arrows, no pill backgrounds:
- `▲ +12%` (up, #1d7a4a)
- `▼ −6pt` (down, #c4353a)
- `— flat` (flat, muted)

## The report-level structure

Every report has this structure:

1. **Report header** (always): logo + divider + "[Brand] × UTM Grabber" tag line
2. **Title block** (always, first section): kicker + italic-accented serif title + metadata row
3. **Content sections**: any combination of section types in any order
4. **Closing section** (always, last): short italic-accented farewell + helper text
5. **Report footer** (always): data source + "Prepared by UTM Grabber · v[version] · [date]"

The header and footer are hardcoded in the template. The title and closing are section types you include in the array.

## Forbidden

- Purple gradient covers, "SECTION N" section labels, "02/05" pagination
- Top accent bars, left accent bars, any colored stripes used for decoration
- Drop shadows, elevation, or any faux-3D effects
- Icons, emoji, or decorative illustrations inside cards
- Multiple italic accents in one title (max one per title)
- Background colors that aren't white or `--b-surface`
- Border radius other than 4px on outer cards (inner elements can be 0 or 4px)
- Font families outside the three approved ones
- Inventing new section types per report

## The one-sentence test

Before shipping any report, look at the output and ask: *"If I put this next to any other report from this skill, would they look like siblings?"*

If yes — ship it. If no — revise.
